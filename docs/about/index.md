# About the ecosystem

The Transmodel Ecosystem is a family of European standards and shared tooling for public transport data. This section covers the things that apply to the whole family — how it's governed, its legal context, the people involved, and how the standards fit together — as distinct from the standards themselves, which each have their own top-level section.

## In this section

- **[Purpose](purpose.md)** — why the ecosystem exists and what problems it solves.
- **[Governance](governance.md)** — CEN, TC 278, WG3, and how European standards get developed.
- **[Legal context](legal-context.md)** — how the standards connect to EU regulations (ITS Directive, MMTIS).
- **[Team](team.md)** — the people currently steering the work.

## Where each standard lives

The individual standards each have their own top-level section — same shape: Overview, Implementations, FAQ, Resources.

- **[Transmodel](../standards/transmodel/index.md)** — the conceptual reference data model that everything else is built on.
- **NeTEx** — scheduled data (timetables, stops, network, calendars, fares, vehicle scheduling).
- **SIRI** — real-time data (vehicle positions, delays, service alerts, facility availability).
- **OJP** — distributed journey planning API.
- **OpRa** — observed and historical operational data.
